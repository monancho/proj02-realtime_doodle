# 유저 플로우

## 핵심 흐름

```txt
[랜딩] -> [Google 로그인] -> [로비] -> [방 만들기/방 입장 모달]
  -> [방 생성] 또는 [방 코드 입력]
  -> [방 대기실]
  -> [게임 시작]
  -> [라운드 진행: 사진 + Canvas + 채팅]
  -> [라운드 결과 저장]
  -> [다음 라운드] 또는 [결과 갤러리]
  -> [결과 이미지 다운로드]
```

## 상태 전이

| 상태 | 진입 조건 | 다음 상태 |
|---|---|---|
| `unauthenticated` | 비로그인 접속 | `authenticated` |
| `authenticated` | 로그인 성공 | `lobby` |
| `lobby` | 로비 진입 | `waitingRoom` |
| `waitingRoom` | 방 생성 또는 입장 성공 | `playing` |
| `playing` | 방장이 게임 시작 | `roundEnding` |
| `roundEnding` | 타이머 종료 | `playing` 또는 `finished` |
| `finished` | 선택 가능한 이미지 없음 | `gallery` |
| `gallery` | 결과 조회 | `download` 또는 `lobby` |

## 주요 시나리오

| 시나리오 | 사용자 행동 | 시스템 반응 |
|---|---|---|
| 방 만들기 | 로그인 후 방 생성 버튼 클릭 | 방 코드 생성, 대기실 이동 |
| 방 참여하기 | 방 코드 입력 후 입장 | 방 유효성 확인 후 참가자 목록 갱신 |
| 사진 업로드 | 파일 선택 후 업로드 | 파일 검증, GridFS 저장, 업로드 현황 갱신 |
| 채팅하기 | 메시지 입력 후 전송 | 같은 방 사용자에게 실시간 표시 |
| 그림 그리기 | 사진 위에 드래그 | 좌표가 Socket.IO로 전파되어 다른 사용자 Canvas에 반영 |
| 라운드 종료 | 타이머 종료 | 드로잉 차단, 결과 이미지 저장 |
| 결과 보기 | 게임 종료 후 갤러리 이동 | 라운드별 결과 이미지 표시 |
| 다운로드 | 다운로드 버튼 클릭 | 이미지 파일 다운로드 |

## 방장 플로우

1. 로그인한다.
2. 로비에서 방 만들기를 선택한다.
3. 방 코드를 참가자에게 공유한다.
4. 참가자와 사진 업로드 현황을 확인한다.
5. 최소 이미지 수가 충족되면 게임 시작을 누른다.
6. 라운드가 자동 진행되는 동안 참여한다.
7. 게임 종료 후 결과 갤러리를 확인한다.

## 참가자 플로우

1. 로그인한다.
2. 방 코드를 입력한다.
3. 대기실에서 사진을 업로드한다.
4. 게임 시작을 기다린다.
5. 라운드 중 채팅과 드로잉에 참여한다.
6. 결과 갤러리에서 이미지를 확인하고 다운로드한다.

## 이탈/재접속 정책

| 상황 | MVP 정책 |
|---|---|
| 대기실에서 socket 이탈 | socket room에서만 퇴장 처리. MVP에서는 MongoDB `rooms.participants`에서 제거하지 않음 |
| 라운드 중 일시 끊김 | HTTP membership이 유지되므로 재접속 시 같은 방 socket room 재입장 허용. 현재 Canvas 복원은 MVP 후순위 |
| 방장 socket 이탈 | 영속 host/participant 정보는 유지. MVP에서는 진행 중 게임 중단 또는 재입장 대기 정책을 후속 단계에서 확정 |
| 업로드 후 이탈 | 이미 업로드된 이미지는 라운드 후보로 유지 |

`participantCount`는 영속 participants 수를 의미한다. 실시간 접속자 수가 필요해지면 후속 단계에서 별도 presence payload 또는 in-memory presence store를 추가한다.

## 와이어프레임 연결

기본 화면 흐름은 `docs/design/assets/realtime-doodle-wireframe-overview.png`를 기준으로 한다. MVP 로그인은 별도 회원가입 화면 없이 Google 로그인으로 시작한다. 백엔드도 Firebase decoded token의 Google provider를 확인해 Google OAuth 사용자가 아닌 token을 거절한다. 로그인 전에는 Google 로그인 진입만 노출하고, 로그인 후 로비에서 `방 만들기`와 `방 입장` 두 가지 CTA만 제공한다. 방 이름과 방 코드는 각각 모달에서 입력한다. 닉네임 변경과 로그아웃은 상단 오른쪽 프로필 메뉴에서 처리한다.
